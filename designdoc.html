Style Guide

Visual Design

Overall, my objective wass to go for a simplistic, modern look. Something thats easy to read and easy on the eye. For that reason i have decided to stick with the Sans Serif font, purely because research has also grading this font one of the best for websites.

I have decided to make majority of my text white and turqoise as my background was darker. Throughout my website i have the same reoccuring transparent image present as the basis for many pages, the white and turqoise creates a nice contrast between this and stands out, without straining the eye. It makes the readability of my website much better.

Once the user would then scroll down thats when they would be met with a white background, here is where i would manipulate my use of semantic markup to create blocks of colour and pictures so that my website doesnt now appear plain. The same turqoise was prominent but ive changed the text not to black as white would hurt my eyes upon reading, as the theme was too bright. I have also added in a fixed nav with a dark green solid background. Since majority of my website consists of three colours, green, black and white i have decided to make the navbar a darker green so when i scroll down, it stands out from the rest and doesnt blend into the existing green and create eye strain. The fixed element also allows for easier access as the user scrolls down.

For my layouts, to best achieve this modern apeal ive stuck with putting as little as possible on the main pages. Ive structured my pages so that the text would mainly sit in the center, better achiving this modern, neat effect.

I have also used bigger font size in this case, so that the users attention would be immediately grabbed by whats in front of them, and where they would like to go. When the text was on the left or right, it would look out of place as i wouldnt right a lot on my main pages, so center is what worked best.

Only later on, as my blog posts and design doc were beigh created is when i made use of a lot of white space, still giving the user some time to breathe by making use of the semantic banners and pictures i mentioned earlier.

I also decided to keep my navbar and logo throughout every page not only because this is what majority of websites have, but because this is the users main way of interaction. They can switch between pages as they please. The secondary form of interaction was my links, and however i created links that would take the user to certain points on he page, especially of that section/page included a lot of information. This made it easier for the user to navigate.

Throughout my website, it remains consistent. The same layout was used throughout. I kept the main pages all similar with that centered effect and as you scrolled down there was always white space mixed with turqoise banners, and sometimes images as backgrounds as can be seen in my blog posts.

Ive created unique pages that still contain the same theme throughout. My biggest issue however was dealing with the design doc as the information overload made it difficult to structure and layout it accordingly. A lot of my structuring here reminded me of powerpoint slides. I have also realized that i hardcode a lot of my texts and images so, when displayed on a different sized monitor it shifts position. I have combated this by adding a fixed nav to the website so even when scrolling down all that information, you can just go back to any page at any time. I was hoping to structure my design doc similarily to how to structures my blog posts, with the accordion but decided against this. Its unique in its own way and there is too much information that if i had also made it ac accordion it would just be one long boring page of white blocks that you can expand.

With the addition of a new blog page, i have decided to keep it short and simple by just creating a coloured background, and adding a white accordion over that. All of my other pages always start off with the transparent image as the background so for this case, since it is an accordion i have just stuck with a plain background so it doesnt look weird switching from image, to plain background behind the accordion. This would also help my websites performance as the old blog page came with a handful of images i set as backgrounds, which i do not have a need for anymore.

In addition my new essays page has also adopted a similar style. In assignment one i have structured it with that same transparent image in the background but since my whole page will be dedicated to one block of text, i have decided the plain greenish blue background with a white box for the text will look better, and aid my website in being more consistent.

Code Interactivity

My UI/UX throughout has remainded consistent and therefore a lot of my code has been the same for certain elements.

I have had to reuse a lot of the same CSS code to create the same effects on different pages, in order to remaim consistent. My CSS styling page is very long, not only because of this but also because i had trouble with the semantic markup within html, i created too many containers and its what made me have to create many different CSS classes etc.

My html was also pretty similar. I originally used a lot of divs and a few articles and sections. However now i have updated my semantic markup and added multiple more like nav/ header/ container as well as reduced the usage of divs. With these groupings i was able to create sections to group large important bodies of text and add visual elements to them.

All of this grouping, while tedious did help me create lot of interesting sections as i was able to manipulate the semantic markup to achieve effects i desired. I did however notice that even though i grouped everything to achieve these effects, they were grouped together very sloppy. I had multiple uneccessary grouping elements within one container, and majority was always just divs. I have changed my semantic markup a lot by not only changing majority of the divs but slightly organizing the grouping so there isnt irrelevant information. I still have a lot of work to do in these aspects as my grouping and CSS can be very confusing.

In addition to this i have also added JS functions to completely restructure my blog posts section. I feel that i have used my functions well in creating an interactive accordion for the users to play around with. I was going to add JS functionality to my favbar, but i couldnt figure out a way to do this without messing up my code so i have just decided to make the navbar fixed. That way when users scroll through long pages such as the design section and blog posts, they can opt out anytime they want.

Responsive Design

My journey

Making my device responsive was a weird journey. I started it off by just changing all of the hard coded values in my CSS to rem. % etc and i thought that this would scale down automatically with the device adjustments. It definetly helped, but my website still looked very weird. There were huge empty spaces when opening it up on a mobile device. I spent my time trying to change this by scaling the boxes but nothing worked. I went to youtube then and remembered that i need to use meda queries!

As you can see by the two examples (iphone SE) I have micely manipulated the content to fit on the page. The fixed navbar and logo remain the same on each page. I have decided to keep the navbar like this because i felt like it would look to plain if i added a menu. The text and links all fit nicely on the main menu. Further down however, i have a bit of spacing problems and shifts within my Design Doc. My main menus for both my blog and essay isnt like the rest as it doesnt all fit nicely on one screen. Another issue i have is that my navbar glitches in the design doc and removed the contacts and design doc link itself on mobile. But none the less, for my first time doing this i dont think it looks that bad. I was only able to manipulate the site to fit well on mobile devices, on ipads and tablets whilst it works slightly, there are still a lot of issues.
